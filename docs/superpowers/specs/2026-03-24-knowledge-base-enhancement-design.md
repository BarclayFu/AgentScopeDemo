# 知识库管理增强设计方案

## 概述

对现有知识库管理页面进行增强，新增多层分类体系、自由标签系统和条目详情弹窗（含知识图谱展示）功能。

## 需求汇总

1. **多层树状分类**：支持任意深度的分类层级（如：法律库 → 民法 → 合同法 → 租赁合同）
2. **多重归属**：一个知识条目可以出现在多个分类路径下
3. **自由标签**：用户可随意添加标签，与分类体系独立
4. **详情弹窗**：点击条目弹出模态框，左侧显示可编辑内容，右侧展示关联知识图谱
5. **即时编辑**：在弹窗内可直接修改标题、内容、分类、标签

## 数据模型

### 方案：分类作为一级实体（多对多关联）

```
┌─────────────┐       ┌──────────────────┐       ┌─────────────┐
│   Category  │──1:N──│ entry_categories │──N:1──│    Entry    │
└─────────────┘       └──────────────────┘       └─────────────┘
       │                                             │
       │                                             │
       │ 1:N                                         │ 1:N
       ▼                                             ▼
┌─────────────┐                               ┌─────────────┐
│   Category  │                               │     Tag     │
│   (self)   │                               └─────────────┘
└─────────────┘                                      │
       ▲                                              │ 1:N
       │                                              ▼
       └────────────────── (via parentId)      ┌─────────────┐
                                              │ entry_tags  │
                                              └─────────────┘
```

### 实体定义

#### Category（分类）
| 字段 | 类型 | 说明 |
|------|------|------|
| id | String | 分类唯一ID |
| name | String | 分类名称 |
| parentId | String | 父分类ID（null 表示根分类） |
| path | String | 完整路径（如 "/法律库/民法/合同法"） |
| level | Integer | 层级深度（0 = 根） |
| createdAt | Long | 创建时间戳 |
| updatedAt | Long | 更新时间戳 |

#### Entry（知识条目）
| 字段 | 类型 | 说明 |
|------|------|------|
| entryId | String | 条目唯一ID |
| title | String | 标题 |
| content | String | 正文内容 |
| source | String | 来源 |
| type | String | 类型 |
| tagIds | List<String> | 关联标签ID列表 |
| createdAt | Long | 创建时间戳 |
| updatedAt | Long | 更新时间戳 |

#### Tag（标签）
| 字段 | 类型 | 说明 |
|------|------|------|
| id | String | 标签唯一ID |
| name | String | 标签名称（如 "重要"、"常用"） |
| createdAt | Long | 创建时间戳 |

#### EntryCategory（条目-分类关联）
| 字段 | 类型 | 说明 |
|------|------|------|
| id | String | 关联ID |
| entryId | String | 条目ID |
| categoryId | String | 分类ID |

## API 设计

### 分类管理
- `GET /api/knowledge/categories` - 获取分类树
- `POST /api/knowledge/categories` - 创建分类
- `PUT /api/knowledge/categories/{id}` - 更新分类
- `DELETE /api/knowledge/categories/{id}` - 删除分类

### 标签管理
- `GET /api/knowledge/tags` - 获取所有标签
- `POST /api/knowledge/tags` - 创建标签
- `DELETE /api/knowledge/tags/{id}` - 删除标签

### 条目管理（扩展现有接口）
- `GET /api/knowledge/entries` - 获取条目列表（支持 categoryId, tagId, search 参数）
- `POST /api/knowledge/entries` - 创建条目（支持 categoryIds, tags）
- `PUT /api/knowledge/entries/{id}` - 更新条目
- `DELETE /api/knowledge/entries/{id}` - 删除条目

### 知识图谱
- `GET /api/knowledge/entries/{id}/graph` - 获取条目关联的知识图谱子图

## 后端改动

### 新增 Service/Repository
- `CategoryService` / `CategoryRepository`
- `TagService` / `TagRepository`
- 扩展 `KnowledgeEntryService`

### 新增 Controller
- `CategoryController` - 分类 CRUD
- `TagController` - 标签 CRUD

### 知识图谱关联
- `GraphRAGRetriever` 已有的 `getSubgraph` 方法可复用
- 需要新增 `EntryGraphController` 提供按条目获取图谱的接口

## 测试要点

1. **分类树**：创建/编辑/删除分类，层级关系正确
2. **条目归属**：同一条目可归属多个分类，删除分类不删除条目
3. **标签**：自由添加/删除标签，标签独立于分类
4. **详情弹窗**：编辑后保存成功，列表同步刷新
5. **图谱展示**：条目详情能正确加载并展示关联图谱

## 实施优先级

1. **Phase 1**：数据模型 + 后端 API（分类、标签 CRUD）
2. **Phase 2**：前端分类树组件 + 标签输入
3. **Phase 3**：条目列表改造 + 分类筛选
4. **Phase 4**：详情弹窗 + 知识图谱集成

## 依赖项

- 复用现有的 `GraphRAGRetriever.getSubgraph()` 方法
- 复用现有的 `KnowledgeGraphView` 中的 Cytoscape 图谱组件
