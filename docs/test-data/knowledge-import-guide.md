# 知识图谱导入指南

## 快速开始

1. 打开前端页面：`/graph/import` 或点击导航栏的"导入数据"
2. 输入知识条目的**标题**和**内容**
3. 点击"预览抽取结果"查看将提取的三元组
4. 确认无误后点击"导入到图谱"

## 支持的格式

### 1. 产品格式
```
产品：智能手表
```
系统会自动识别"智能手表"为产品实体。

### 2. 服务关键词
内容中包含以下关键词会被识别为服务：
- 保修、质保、维修
- 退换、退款、退货、换货

### 3. 问答格式
```
智能手表如何保修？
答案：请联系客服处理。
```
或：
```
智能手表如何保修？
答：请联系客服处理。
```

### 4. 组合格式
支持在一个知识条目中组合使用以上格式：

```
产品：智能手表
保修：两年质保

智能手表如何保修？
答案：请联系客服处理。

退换货政策：收货7天内可申请退换货。
```

## 预览结果说明

点击"预览抽取结果"后，你会看到：

### 识别的实体
显示系统从内容中识别到的：
- **products**: 产品名称列表
- **services**: 服务关键词列表
- **qas**: 问答对列表

### 规则抽取
基于正则表达式抽取的三元组，例如：
- `(智能手表, MENTIONS, 智能手表如何保修？)`
- `(智能手表如何保修？, HAS_SERVICE, 两年质保)`
- `(智能手表如何保修？, RELATED_TO, 请联系客服...)`

### LLM抽取
基于大语言模型抽取的深层语义三元组。
> 注意：LLM抽取需要配置 MiniMax API Key，如果没有配置则不会返回LLM抽取结果。

## 验证导入结果

导入成功后，可以：

1. 点击"查看图谱"按钮，跳转到 `/graph` 页面查看可视化图谱
2. 在图谱中确认节点和关系是否正确
3. 在对比实验页面测试检索效果

## 故障排除

### 预览显示"共 0 个三元组"
可能原因：
- 内容格式不符合预期
- LLM抽取失败（需要检查API Key配置）

解决方法：确保内容中包含明确的产品、服务或问答格式。

### 导入后图谱为空
可能原因：
- Neo4j服务未启动
- 实体类型推断失败（实体的类型推断依赖于内容中是否包含明确的实体关键词）

解决方法：检查Neo4j服务状态，并确保内容格式正确。
