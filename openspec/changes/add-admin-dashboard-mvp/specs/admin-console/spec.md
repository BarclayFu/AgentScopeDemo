## ADDED Requirements

### Requirement: 管理后台仪表盘
系统 MUST 提供一个可用的管理后台仪表盘页面，用于展示客服系统当前运行摘要。

#### Scenario: 成功加载仪表盘指标
- **WHEN** 管理员打开仪表盘页面
- **THEN** 前端调用监控摘要接口并展示活跃会话数、消息总数、工具调用数、错误数和平均响应时间

#### Scenario: 仪表盘刷新状态
- **WHEN** 管理员手动刷新或页面自动刷新
- **THEN** 页面更新最新指标并显示最近一次检查时间

#### Scenario: 仪表盘接口异常
- **WHEN** 监控接口请求失败
- **THEN** 页面显示错误提示且保留可重试入口

### Requirement: 结构化监控接口
系统 MUST 为管理后台提供结构化 JSON 的监控摘要和健康状态接口。

#### Scenario: 获取监控摘要
- **WHEN** 前端调用监控摘要接口
- **THEN** 服务返回包含摘要指标和检查时间的 JSON 对象

#### Scenario: 获取服务健康状态
- **WHEN** 前端调用健康状态接口
- **THEN** 服务返回包含服务名称、状态和值守时间的 JSON 对象

#### Scenario: 重置监控统计
- **WHEN** 管理员调用监控重置接口
- **THEN** 服务重置统计信息并返回明确的 JSON 结果

### Requirement: 持久化操作设置
系统 MUST 持久化管理后台中的前端操作设置，并在聊天页面中自动应用。

#### Scenario: 保存默认聊天设置
- **WHEN** 管理员在设置页修改默认用户 ID、流式速度或 API 地址并保存
- **THEN** 设置被写入本地存储并在刷新页面后继续生效

#### Scenario: 聊天页读取默认设置
- **WHEN** 用户打开聊天页面
- **THEN** 页面自动读取已保存的默认用户 ID、流式速度和 API 地址

#### Scenario: 恢复默认设置
- **WHEN** 管理员在设置页执行恢复默认值
- **THEN** 本地存储中的设置被重置为项目默认值
