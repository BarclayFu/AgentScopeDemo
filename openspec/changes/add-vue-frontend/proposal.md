# 变更：创建Vue.js前端项目替换现有静态页面

## 为什么
现有前端是两个简单的静态HTML页面，功能单一，缺乏扩展性：
- 代码无法复用
- 无法方便地添加新页面和功能
- 维护性差
- 用户体验有限

使用Vue.js框架可以提供更好的开发体验和扩展性。

## 什么变化
- 创建独立的Vue.js前端项目
- 实现聊天功能页面（保留流式输出）
- 预留路由和组件结构，支持未来扩展
- 与现有后端API对接
- 替换现有的静态HTML页面

## 影响
- 受影响的规范：前端模块
- 受影响的代码：
  - 新增：`frontend/` 目录（Vue项目）
  - 保留：`customer-service-agent/src/main/resources/static/`（作为静态资源备用）

## 关键约束
- 必须与现有后端API完全兼容
- 需要支持SSE流式输出
- 需要预留足够的扩展空间
- 保持用户体验与现有流式页面相当

## 未来扩展预留
- 用户管理页面
- 知识库管理页面
- 数据统计/仪表盘
- 系统设置
